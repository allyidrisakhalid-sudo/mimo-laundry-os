# Metadata Map  Mimo Phase 2

## 1) Metadata Principles

- every public route must have one clear metadata purpose
- metadata must match visible page truth
- titles and descriptions must stay route-specific
- no duplicate title or description intent across public routes
- metadata must remain calm, premium, and direct
- auth and legal metadata must stay useful without spam
- canonical ownership must be explicit, not improvised later
- production metadata must resolve to one host only:
  - https://mimolaundry.org

## 3) OG Image Family Rules

- one branded OG system
- Midnight Silk visual identity
- readable title area
- minimal clutter
- premium dark/light balance
- no route-specific chaotic art direction

Additional OG family discipline:
- use one restrained template family across all public routes
- preserve legibility on mobile share cards
- keep route text short enough to remain readable
- maintain premium dark anchors, silk-white reading surfaces, and restrained accent use
- legal and utility routes may use simplified variants but must stay within the same family

## 4) Canonical Enforcement Rules

- production URL only
- absolute URLs only
- no query canonicals
- consistent trailing slash handling
- no duplicate host variants

Canonical enforcement detail:
- root canonical is https://mimolaundry.org/
- non-root public routes use no trailing slash style
- www and non-production hosts must not publish alternate canonicals
- route-level canonicals must match metadata ownership exactly
- /track parameter states must resolve to the single canonical route, not many crawlable variants

## 5) Metadata Lock Statement

The metadata system is locked to a route-specific, premium, minimal structure for the public Mimo site. Each required public and legal route has one canonical URL, one clear metadata purpose, and one consistent OG image family under the Midnight Silk direction. This prevents duplication, social-preview drift, and future ad hoc metadata decisions.
