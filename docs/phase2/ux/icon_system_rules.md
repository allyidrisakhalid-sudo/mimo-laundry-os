# Icon System Rules  Mimo Phase 2

## 1) Chosen Icon System

- Icon family: Lucide
- Style direction: rounded outline icons only
- Stroke feel: clean, modern, medium-light, premium
- Usage scope: public pages, customer app, driver app, hub portal, affiliate portal, admin portal, dev portal

## 2) Why This System Fits Mimo

Lucide fits Mimo because it is minimal, modern, readable at small sizes, consistent across interfaces, and compatible with the premium Midnight Silk direction without looking playful or enterprise-heavy.

## 3) Icon Style Rules

- outline icons only by default
- do not mix filled, duotone, emoji, and outline styles
- rounded consistent visual language
- icons support labels, not replace labels
- decorative icon overuse is forbidden
- one icon metaphor per concept only

## 4) Size Rules

- 16px: dense metadata or inline label support
- 18px: standard button/icon-with-label usage
- 20px: nav items and cards
- 24px: feature highlights / empty states / hero support
- 32px+: rare, only for brand moments and large KPI visual anchors

## 5) Weight and Alignment Rules

- keep default Lucide stroke consistency
- do not resize randomly inside the same component type
- align icons optically with text baselines
- icon containers may use soft tinted or gradient-backed circles only in highlighted cards
- standard nav and button icons must stay plain

## 6) Functional Usage Rules

### Navigation
- navigation icons are supportive and secondary to text

### Status
- status icons must reinforce state, not replace wording

### Actions
- destructive actions require clear label plus icon if used

### Empty States
- empty states may use one large icon plus message

### Dashboard KPIs
- KPI cards may use one icon only

### Alerts and Exceptions
- alerts use icon + text + color together

## 7) Forbidden Icon Patterns

- mixing icon families
- using emojis as icons
- using random illustrated stickers
- using filled icons in one menu and outline in another
- rotating or stretching icons decoratively
- using icons without labels for critical actions
- colorizing icons outside brand/semantic rules

## 8) Icon Lock Statement

### P2.1 Icon Lock

All future Phase 2 interfaces must use Lucide as the sole icon family with these rules.
