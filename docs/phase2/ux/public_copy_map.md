# Public Copy Map  Mimo Phase 2

## 1) Copy Principles

Locked copy principles:
- warm
- premium
- short
- direct
- mobile-first scanning
- no fluff
- no hardcoded final UI strings outside i18n later
- English and Swahili parity must remain possible

Additional copy guardrails:
- every page must explain itself quickly
- headlines must describe the page purpose clearly
- support copy must stay compact and operationally useful
- CTA wording must be action-led and calm
- repeated language across routes must be intentional, not lazy duplication

## 4) Repetition Guard

Locked repetition rules:
- start ordering lives on home/login/signup entry path
- tracking lives on /track
- support lives on /help plus structured WhatsApp placements
- partner recruitment lives on /partners
- policy detail lives only on legal pages
- no page should duplicate another pages full function

Additional guardrails:
- home may reference support, but does not become the help center
- help may reference tracking, but does not duplicate the tracking form as its main function
- partners may mention onboarding, but does not duplicate login/signup flows
- legal pages may explain policies, but do not become FAQ pages
- footer links support discovery, not page duplication

## 5) Public Copy Lock Statement

The public copy model is locked to a short, premium, mobile-first, non-duplicative system that can be implemented with full EN/SW parity. Each route has one clear purpose, one main action, and tightly controlled supporting language so the public experience stays calm, fast to understand, and consistent with the approved Midnight Silk direction.
