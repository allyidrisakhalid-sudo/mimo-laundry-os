# Hub Tablet Behavior Spec  Mimo Phase 2

## 1) Tablet/Web Principles

Locked tablet/web principles:
- repeated action speed
- scanner-first where relevant
- touch-friendly on tablet
- dense enough for operations, never cramped
- queues and boards readable at a glance
- no desktop micro-controls
- no mobile-style oversimplification that slows staff down
- portal must work cleanly on tablet and web

## 2) Dashboard Tablet/Web Behavior

- quick routing into active queue
- KPI row visible without dominating
- action queue remains obvious

## 3) Intake Tablet/Web Behavior

- scan/input zone immediately reachable
- queue and active item may coexist on larger screens
- controls large enough for touch and fast keyboard/scanner use

## 4) Processing Board Tablet/Web Behavior

- columns readable without chaos
- cards easy to scan
- issue flags and SLA chips visible from a standing distance
- stage movement remains easy without pixel-precision drag

## 5) Order Detail and QC Tablet/Web Behavior

- context and action stay together
- QC pass/fail remains quick
- issue state visible without extra navigation layers

## 6) Dispatch Tablet/Web Behavior

- zone filter and driver selection visible together
- repeated assignments remain fast
- selection flow does not become checkbox clutter

## 7) Performance and Repetition Rules

- high-frequency repeated actions must stay lightweight
- no unnecessary animation delays
- no clutter from oversized decorative surfaces
- intake and processing screens must feel operationally fast all day

## 8) Tablet/Web Lock Statement

The hub portal is locked for clean tablet and web operations. Scanner-first intake, readable stage boards, focused order detail, and fast dispatch assignment are all optimized for repeated operational use without desktop-only precision patterns or mobile-style simplification that slows staff down.
