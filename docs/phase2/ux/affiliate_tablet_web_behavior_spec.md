# Affiliate Tablet/Web Behavior Spec  Mimo Phase 2

## 1) Tablet/Web Principles

Locked tablet/web principles:
- fast counter operation
- repeated use during busy periods
- touch-friendly on tablet
- simple enough for quick staff turnover
- no hidden next steps
- low-cognitive-load order handling
- finance remains simple and non-distracting
- portal must work cleanly on tablet and web

## 2) Dashboard Behavior

- next action visible immediately
- create-order CTA dominant
- readiness and issues easy to scan
- KPI row must stay concise and readable on desktop and tablet widths
- action queue must surface ready items and issues without forcing deep navigation first
- support panel must remain visible but secondary to operational actions
- affiliate admin finance snapshot appears only as a compact summary card area and never displaces order actions

## 3) Order Creation Behavior

- one-page fast form preferred
- fields grouped clearly
- submit action always obvious
- repeated order entry remains fast
- customer details should appear first and remain lightweight
- service and return/handoff controls must stay visible without long vertical scanning
- review summary should stay short and high-confidence near the submit area
- controls must be large enough for repeated tablet tapping during busy shop periods

## 4) Orders List and Detail Behavior

- list remains compact and scoped
- detail gives full shop truth without clutter
- order readiness and handoff state visible quickly
- filters remain limited to the few states that matter for daily shop work
- rows/cards must be touch-friendly and readable without admin-grid density
- detail page must place readiness, next action, and status progression above secondary metadata
- no cross-shop hints, toggles, or empty structural placeholders should appear anywhere in the list or detail views

## 5) Pickup Workflow Behavior

- pickup-ready state easy to scan
- confirmation action large enough for fast use
- issue path visible but secondary
- the valid next action must be visually dominant when pickup or handoff is possible
- issue/help entry must remain present without overpowering the main operational path
- state change after pickup confirmation should be immediate and clearly reflected in the page

## 6) Finance Behavior

- finance page clean and summary-based
- no oversized charts
- recent payout list remains readable
- affiliate staff never sees finance navigation
- summary cards must communicate earned, pending, and recent payout status at a glance
- the supporting list/table should remain practical, low-density, and easy to scan on tablet and desktop

## 7) Performance and Repetition Rules

- daily repeated actions must stay lightweight
- no over-animation
- no admin-style density
- portal should support fast shop work all day
- skeleton/loading states must preserve layout stability
- repeated order creation, list review, and pickup confirmation should not require extra detours
- interactions must favor calm speed over decorative complexity

## 8) Tablet/Web Lock Statement

The affiliate portal tablet/web behavior is locked as a clean, touch-friendly, low-cognitive-load operational system for busy shop use. Dashboard awareness, fast order entry, scoped order management, clear pickup handling, and simple finance visibility all work together without admin density, cross-shop confusion, or hidden next steps.
