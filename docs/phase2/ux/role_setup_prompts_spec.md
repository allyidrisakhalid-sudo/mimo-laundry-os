# Role Setup Prompts Spec  Mimo Phase 2

## 1) Setup Prompt Principles

Locked setup prompt principles:
- operator onboarding must happen inside the portal home
- setup prompts must be card-based, not tutorial-based
- one prompt card per role at a time
- setup prompt must point to real setup actions only
- no training content wall
- no slideshow
- no empty ceremonial checklists
- prompt must explain what to do next in plain language
- once complete, prompt must disappear or downgrade naturally

## 2) Driver First-Run Prompt

### Trigger
- first valid driver portal entry when profile/availability/basic readiness is incomplete

### Placement
- top of /app/driver home above the main queue

### Card Title Intent
- complete your setup

### Purpose
- make the driver ready to receive and complete tasks

### Required Checklist Items
- confirm availability
- confirm phone/profile details if editable
- understand proof/cash expectation in one short line

### Primary Action
- Complete setup

### Rules
- no training deck
- no separate help course
- one short explanatory line only
- after completion, driver lands back in Today view with next task clarity

## 3) Hub Staff First-Run Prompt

### Trigger
- first valid hub portal entry when key working context is incomplete or needs acknowledgment

### Placement
- top of /app/hub dashboard

### Purpose
- orient hub staff to the intake  processing  dispatch workflow in one short setup block

### Required Checklist Items
- confirm working hub context if needed
- understand the three queue sequence
- know where exceptions appear

### Primary Action
- Complete setup

### Rules
- use one short checklist card
- no deep onboarding flow
- no duplicate SOP wall
- once completed, return to dashboard with the next queue obvious

## 4) Affiliate Staff First-Run Prompt

### Trigger
- first valid affiliate portal entry when user has not yet confirmed the core shop-order workflow

### Placement
- top of /app/affiliate dashboard

### Purpose
- orient shop staff to create order, manage orders, and hand off/return clearly

### Required Checklist Items
- create order path
- orders list purpose
- handoff/readiness awareness

### Primary Action
- Complete setup

### Rules
- one concise card
- no payout/admin content in affiliate staff prompt
- no training article wall

## 5) Affiliate Admin First-Run Prompt

### Trigger
- first valid affiliate admin portal entry when setup acknowledgment has not been completed

### Placement
- top of /app/affiliate dashboard

### Purpose
- orient affiliate admin to operations plus commission/payout visibility

### Required Checklist Items
- create/manage orders
- track readiness/issues
- review finance area

### Primary Action
- Complete setup

### Rules
- same card family as affiliate staff
- include finance visibility note
- still remain concise and non-training-based

## 6) Prompt Completion Rules

- completing setup removes the high-priority first-run card
- completed users continue normal dashboard usage
- setup completion must not create hidden dependencies on a long tutorial sequence
- the dashboard must immediately expose the next real operational action after completion

## 7) Prompt Reappearance Rules

- prompt should not reappear after true completion
- prompt may reappear only if the system still detects required missing readiness data
- do not nag users repeatedly for already completed setup

## 8) Prompt Lock Statement

The operator first-run model for Phase 2 is locked to concise dashboard-native setup prompts only. Driver, hub, affiliate staff, and affiliate admin users receive one role-correct setup card that points to real readiness actions and then disappears once genuinely complete. No training modules, duplicated SOP walls, or ceremonial onboarding layers are allowed in this chapter.
