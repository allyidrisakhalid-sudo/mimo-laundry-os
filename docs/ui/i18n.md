# Internationalization (EN/SW)

## Library choices

- Web: `i18next` + `react-i18next`
- Mobile: `i18next` + `react-i18next`
- Web preference persistence: browser localStorage through i18next browser language detector
- Mobile preference persistence: AsyncStorage

## Key naming convention

Use stable hierarchical keys:

- `common.ok`
- `common.cancel`
- `nav.home`
- `auth.login.title`
- `order.status.out_for_delivery`

Rules:

- No inline English in keys
- Keep keys stable even if display copy changes
- Business copy lives in app translation files, not `packages/ui`

## Translation file locations

### Web

- `apps/web/src/i18n/en.json`
- `apps/web/src/i18n/sw.json`

### Mobile

- `apps/mobile/src/i18n/en.json`
- `apps/mobile/src/i18n/sw.json`

## Adding new translations

1. Add the same key to both EN and SW files
2. Use `t("your.key")` in the screen/component
3. Do not hardcode user-facing strings in UI screens
4. Keep names grouped by feature area

## Formatting helpers

Shared helpers live in `packages/types/src/formatters.ts`:

- `formatMoneyTzs(amount)`
- `formatDateTime(date)`
- `formatPhoneTz(phone)`

Examples:

- `formatMoneyTzs(45000)` -> `TZS 45,000`
- `formatDateTime(new Date("2026-01-29T14:30:00"))`
- `formatPhoneTz("0712345678")` -> `+255 712 345 678`
